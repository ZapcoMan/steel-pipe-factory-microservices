# 钢铁工厂小程序 - 开发文档

## 📋 项目概述

钢铁工厂小程序是一个专为钢铁行业设计的微信小程序，提供生产管理、数据分析、计算工具和技术资料查询等功能。

### 技术栈
- **框架**: Vue 3 + UniApp
- **语言**: TypeScript
- **UI**: 自定义组件库（使用 rpx 单位适配）
- **状态管理**: Vue 3 Composition API
- **图表**: uCharts (@qiun/ucharts)

## 🔌 API 接口详解

### 1. 基础配置

#### 配置文件位置
- **主配置**: `src/config/index.ts`
- **环境变量**: `.env` (开发), `.env.production` (生产)

#### 环境配置参数
```typescript
{
  VITE_API_BASE_URL: 'http://192.168.1.12:9990',  // API 基础地址
  VITE_TIMEOUT: 60000                              // 请求超时时间 (毫秒)
}
```

### 2. HTTP 请求封装

**文件位置**: `src/utils/request.ts`

#### HttpRequest 类方法

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `request<T>` | 通用请求方法 | RequestOptions | Promise<ResponseData<T>> |
| `get<T>` | GET 请求 | url, params? | Promise<ResponseData<T>> |
| `post<T>` | POST 请求 | url, data? | Promise<ResponseData<T>> |
| `put<T>` | PUT 请求 | url, data? | Promise<ResponseData<T>> |
| `delete<T>` | DELETE 请求 | url | Promise<ResponseData<T>> |
| `setHeaders` | 设置全局请求头 | headers | void |

#### 请求拦截器功能
- 自动添加 `Content-Type: application/json`
- 支持自定义请求头
- 可添加 Token 认证（已注释）

#### 响应拦截器功能
- **200/201**: 成功，直接返回数据
- **401**: 未授权，提示重新登录
- **403**: 拒绝访问
- **404**: 资源不存在
- **500**: 服务器错误
- **其他**: 请求失败提示

## 🔧 开发指南

### 1. 环境搭建

```bash
# 安装依赖
pnpm install

# 启动开发模式（微信小程序）
pnpm run dev:mp-weixin

# 构建生产版本
pnpm run build:mp-weixin
```

### 2. API 地址配置

#### 开发环境
编辑 `.env` 文件：
```env
VITE_API_BASE_URL=http://192.168.1.12:9990
VITE_TIMEOUT=60000
```

#### 生产环境
编辑 `.env.production` 文件：
```env
VITE_API_BASE_URL=https://api.steelfactory.com
VITE_TIMEOUT=60000
```

### 3. 添加新 API 接口

#### 步骤 1: 在对应 API 文件中添加方法
```typescript
// src/api/xxxApi.ts
class XxxApiService {
  getNewData() {
    return http.get('/uniapp/getNewData').then((res: ResponseData) => {
      if (res && res.code === 20000) {
        return res.data;
      }
      // 添加容错处理
    }).catch(() => {
      // 返回模拟数据或错误处理
    });
  }
}
```

#### 步骤 2: 在页面中调用
```typescript
import xxxApi from '@/api/xxxApi';

const loadData = async () => {
  try {
    const data = await xxxApi.getNewData();
    // 处理数据
  } catch (error) {
    console.error('获取数据失败:', error);
    uni.showToast({
      title: '加载失败',
      icon: 'none'
    });
  }
};
```

### 4. 容错处理规范

所有 API 调用都应包含容错机制：
1. **网络错误处理**: 使用 try-catch
2. **数据验证**: 检查返回数据的 code
3. **用户提示**: 使用 uni.showToast
4. **降级方案**: 准备模拟数据作为备选

- 变量名使用 **camelCase**
- TypeScript 严格模式
- 所有 API 方法必须有类型注解

## ⚠️ 注意事项

### 1. API 调用规范
- 所有 API 请求都经过统一的 request 封装
- 必须处理异常情况
- 建议添加 loading 提示

### 2. 环境变量
- 以 `VITE_` 开头的环境变量才会暴露给客户端
- 修改环境变量后需重启开发服务器

### 3. 图片资源
- 本地图片放在 `static/` 目录
- 使用绝对路径引用

### 4. 小程序限制
- 注意小程序包大小限制
- 合理使用分包加载（如需要）
